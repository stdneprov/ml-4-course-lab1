# Отчет по лабораторной работе
## Исследование моделей семантической сегментации огня на изображениях

### 1. Выбор начальных условий
#### 1.1 Выбор набора данных
В данной работе использован набор данных для семантической сегментации огня (`Fire Segmentation Dataset`). Данные представляют собой изображения, содержащие участки огня или его имитации, и соответствующие им бинарные маски, где пиксели огня выделены белым цветом, а фон — черным.

**Структура датасета:**
- `fire/images/fire` — исходные изображения с огнем
- `fire/masks` — бинарные маски сегментации огня

**Обоснование выбора набора данных:**
Автоматическое обнаружение и сегментация огня является критически важной задачей для систем видеонаблюдения, пожарной безопасности, мониторинга промышленных объектов и автономных роботизированных систем. Данный датасет выбран по следующим причинам:
- Задача имеет высокую практическую значимость и требует точного выделения объектов сложной формы и текстуры.
- Данные содержат выраженный дисбаланс классов (огонь занимает значительно меньшую площадь, чем фон), что создает вызов для моделей и требует подбора специфичных функций потерь.
- Структура датасета позволяет исследовать влияние аугментаций и различных архитектур на устойчивость сегментации.

#### 1.2 Выбор метрик качества
Для оценки качества моделей семантической сегментации в данной работе использованы следующие метрики:
- **Dice Score** вычисляется по формуле `2 * TP / (2 * TP + FP + FN)`. Метрика оптимальна для несбалансированных данных, так как акцентирует внимание на перекрытии предсказанной и истинной областей.
- **IoU (Intersection over Union)** вычисляется по формуле `TP / (TP + FP + FN)`. Стандартная метрика в задачах сегментации, жестко штрафующая за ложные срабатывания и пропуски.
- **Precision** вычисляется по формуле `TP / (TP + FP)`. Важна для минимизации ложных обнаружений огня.
- **Recall** вычисляется по формуле `TP / (TP + FN)`. Важна для полноты обнаружения всех участков огня на изображении.

Комбинация этих четырех метрик позволяет получить объективную картину качества модели, учитывая как точность границ, так и баланс между ложноположительными и ложноотрицательными предсказаниями.

### 2. Создание бейзлайна и оценка качества
#### 2.1 Выбор моделей
Для создания бейзлайна использованы две готовые архитектуры из библиотеки `segmentation_models_pytorch`:
- **CNN Baseline**: U-Net с энкодером `ResNet34` (предобученным на ImageNet).
- **ViT Baseline**: U-Net с энкодером `mit_b0` (MixTransformer, предобученным на ImageNet).

#### 2.2 Условия обучения
Обучение проводилось при следующих параметрах:
- количество эпох: 6
- размер батча: 8
- оптимизатор: Adam
- learning rate: 3e-3
- функция потерь: DiceLoss (`from_logits=True`) для CNN, комбинированная Dice+BCE для ViT
- размер изображения: 256x256 пикселей
- оценка проводилась на обучающей выборке для сравнения динамики обучения

#### 2.3 Результаты бейзлайна
Результаты обучения моделей на исходных данных:
- **Модель CNN Baseline** показала следующие результаты: `Dice: 0.6844 | IoU: 0.5244 | Precision: 0.6970 | Recall: 0.6796`
- **Модель ViT Baseline** показала следующие результаты: `Dice: 0.5856 | IoU: 0.4176 | Precision: 0.5977 | Recall: 0.5839`

### 3. Улучшение бейзлайна
#### 3.1 Сформулированные гипотезы
Для улучшения качества моделей были сформулированы следующие гипотезы:
1. **Гипотеза 1**: Применение расширенных аугментаций данных (отражения, повороты, сдвиги, масштабирование, изменение яркости/контраста и гамма-коррекция) повысит устойчивость моделей к вариативности входных изображений.
2. **Гипотеза 2**: Использование оптимизатора AdamW с уменьшенным learning rate обеспечит лучшую регуляризацию и более стабильную сходимость.
3. **Гипотеза 3**: Добавление механизма внимания `scSE` в декодер улучшит фокусировку модели на релевантных признаках огня.
4. **Гипотеза 4**: Использование комбинированной функции потерь Dice + BCE позволит учесть дисбаланс классов и обеспечит более стабильные градиенты на ранних этапах обучения.

#### 3.2 Улучшенный бейзлайн
На основе проверки гипотез сформирован улучшенный пайплайн обучения:
- **Аугментации**: `HorizontalFlip`, `VerticalFlip`, `RandomRotate90`, `ShiftScaleRotate`, `RandomBrightnessContrast`, `RandomGamma`
- **Оптимизатор**: AdamW (`lr=1e-4` для CNN, `lr=1e-3` для ViT)
- **Функция потерь**: комбинированная Dice + BCE
- **Архитектура**: для ViT дополнительно добавлен механизм внимания `scSE` в декодер и расширены каналы декодера

#### 3.3 Результаты улучшенного бейзлайна
Обучение улучшенного бейзлайна дало следующие результаты:
- **Модель CNN Enhanced**: `Dice: 0.7960 | IoU: 0.6636 | Precision: 0.8016 | Recall: 0.7940`
- **Модель ViT Enhanced**: `Dice: 0.3104 | IoU: 0.1889 | Precision: 0.5974 | Recall: 0.2169`

#### 3.4 Сравнение результатов
Сравнение улучшенного бейзлайна с исходным показало следующее:
- Для модели **CNN** получилось добиться значительного улучшения точности (+0.1116 по Dice), что подтверждает эффективность аугментаций и AdamW для сверточных архитектур.
- Для модели **ViT** наблюдается существенное ухудшение результата. Вероятные причины: слишком низкий learning rate в сочетании со сложными аугментациями, изменение функции потерь, либо недостаточное количество эпох для сходимости трансформерной архитектуры с дополнительным вниманием. Требуется увеличение количества эпох или тонкая настройка гиперпараметров.

### 4. Имплементация алгоритмов машинного обучения
#### 4.1 Самостоятельная имплементация моделей
В рамках работы были самостоятельно имплементированы две архитектуры:
- **Сверточная архитектура U-Net**: классическая структура с блоками `DoubleConv`, `MaxPool`, `Upsample` и skip-connections.
- **Трансформерная архитектура SimpleViTSegmenter**: разбивка изображения на патчи, линейное встраивание, добавление позиционного кодирования, пропускание через `TransformerEncoder`, обратное преобразование в пространственную карту и билинейное восстановление до исходного разрешения.

#### 4.2 Обучение имплементированных моделей
Обучение собственных моделей проводилось с использованием оптимизатора Adam (`lr=3e-3`), функции потерь `BCEWithLogitsLoss` и базовых аугментаций.
- **Собственная модель CNN** показала следующие результаты: `Dice: 0.6632 | IoU: 0.5006 | Precision: 0.8040 | Recall: 0.5709`
- **Собственная модель ViT** показала следующие результаты: `Dice: 0.2341 | IoU: 0.1356 | Precision: 0.1512 | Recall: 0.9652`

#### 4.3 Добавление техник улучшенного бейзлайна
К собственным моделям были применены техники из улучшенного пайплайна:
- расширенные аугментации данных
- оптимизатор AdamW (`lr=1e-3`)
- комбинированная функция потерь Dice + BCE (для ViT Enhanced)

#### 4.4 Обучение улучшенных собственных моделей
После добавления техник улучшенного бейзлайна собственные модели показали следующие результаты:
- **Модель Own CNN Enhanced**: `Dice: 0.6690 | IoU: 0.5071 | Precision: 0.7892 | Recall: 0.5877`
- **Модель Own ViT Enhanced**: `Dice: 0.4318 | IoU: 0.2803 | Precision: 0.5047 | Recall: 0.3889`

Собственные архитектуры демонстрируют стабильную, но несколько меньшую производительность по сравнению с библиотечными аналогами, что ожидаемо из-за отсутствия предобученных весов энкодера и менее оптимизированной реализации. Однако применение улучшенных техник позволило заметно повысить качество собственной ViT-модели.

### 5. Заключение
В ходе выполнения лабораторной работы были решены следующие задачи:
1. Выбран и обоснован набор данных для семантической сегментации огня. Выбраны и обоснованы метрики качества: Dice, IoU, Precision, Recall.
2. Создан бейзлайн с использованием библиотечных моделей на базе CNN и Transformer. Проведена оценка качества обучения.
3. Сформулированы и проверены гипотезы по улучшению качества. Сформирован улучшенный пайплайн с расширенными аугментациями, AdamW, механизмами внимания и комбинированной функцией потерь.
4. Обучены модели с улучшенным бейзлайном. Показано, что CNN-архитектуры эффективно реагируют на улучшения, тогда как Transformer-модели требуют более тщательной настройки гиперпараметров и большего числа эпох.
5. Самостоятельно имплементированы сверточная (U-Net) и трансформерная (SimpleViTSegmenter) модели.
6. Добавлены техники улучшенного бейзлайна к собственным моделям, что подтвердило универсальность предложенных методов регуляризации и аугментации.

**Итоговый вывод:** Для задачи сегментации огня на изображениях оптимальным выбором в условиях ограниченных вычислительных ресурсов и данных является архитектура U-Net с предобученным CNN-энкодером (ResNet34), дополненная AdamW и комбинированной функцией потерь. Трансформерные подходы демонстрируют потенциал, но требуют большего объема данных, более длительного обучения и тонкой настройки для достижения стабильных результатов. В дальнейших экспериментах рекомендуется внедрить строгое разделение на train/val/test и использовать раннюю остановку (Early Stopping) для предотвращения переобучения.
